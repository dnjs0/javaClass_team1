 자바는 객체 지향 프로그래밍 언어다.
객체를 중심으로 프로그램을 만드는 방식

클래스라는 설계도를 사용해서 객체라는것을 만들고 만들어진 객체를 사용해서 내가 원하는 행동을 구현 하는 것

클래스는 배열과 비슷한 구조를 가지고 있지만 차이점은 배열은 번호로 지정하고 클래스는 이름으로 지정한다.

for 문을 사용 할것 같으면 배열을 사용하고, 아니면 클래스를 사용한다.

클래스는 비용이 많이 들지만 데이터 저장 관리가 용이하다.

설계한 클래스로부터 객체를 만드는 방법은
자료형    변수명 =  new                생성자();
클래스명 객체명 = 객체생성연산자 생성자(); 로 만든다.


객체는 인스턴스와 거의 유사한 언어라고 할수 있다.


클래스를 선언할때 클래스명은 파스칼 표기법으로 한다.
클래스를 설계할때 객체의 속성과 객체의 행동을 정해줘야하는데 속성은 변수이고 행동은 메서드를 선언한다.


ex31
클래스를 선언하는 방법
1. 클래스 선언하기 
2. 멤버 선언하기
	a. 변수
 	b. 메서드

클래스를 선언할 때 권장사항
- 클래스 1개당 파일을 하나만 만드는것이 좋다. 파일이 저장될때 대표클래스(public) 이름으로 저장이 되기 때문에 그안에 선언된 클래스를 직관적으로 찾을수 없기 때문이다.

멤버 변수 > 객체의 상태(속성) property
멤버 메서드(= 메서드) > 객체의 행동 -  좋은메서드. 자신만의 특성이 있는 행동. [반드시! 자신의 상태(멤버 변수)를 활용해서 구현]

OOO 지정자(제어자)
클래스 or 클래스 멤버 > 부가적인 기능을 제공하는 키워드(옵션)

접근 지정자(제어자), Access Modifier  ex) 경비아저씨, 보안요원
클래스 or 클래스 멤버에 적용한다.

1. public (100% 공개)      2. private(100% 비공개)

멤버가 소속된 클래스 영역을 기준으로 > 경계선/담벼락이다.

ex)public 리모컨의 버튼  private 리모컨의 케이스  캡슐화
접근 제한을 두는 이유 예시 : 사람의 피부가 없다면 누군가 와서 심장을 조물딱 하기 가능.

접근 지정자 가이드
1. 멤버 변수는 무조건 private으로 선언한다.
2. 멤버 메서드를 선언한다.

메서드를 선언하면 클래스 내에서 흐름을 만들어 줄수 있다. (값이 바로 출력이 되는것을 통제할수 있다.

메서드안에서 지역변수를 지정할때 멤버변수와 이름이 같다면 서로 충돌이 일어날수 있기 때문에 메소드 안에 멤버변수를 선언할때는 this. 연산자를 사용한다.
